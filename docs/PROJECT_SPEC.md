# Προδιαγραφές — Διαχείριση Πολυκατοικίας

## Σκοπός
Εφαρμογή για τον διαχειριστή πολυκατοικίας: καταγραφή δαπανών, επιμερισμός
κοινοχρήστων κατά χιλιοστά, πληρωμές/υπόλοιπα, έκδοση & εκτύπωση καταστάσεων,
με μελλοντικό portal ενοίκων (συζήτηση, ψηφοφορίες, ανακοινώσεις).

## Ρόλοι
- **admin / manager** — πλήρης διαχείριση.
- **owner** — καρτέλα, υπόλοιπα, (μελλοντικά) ψήφος & συζήτηση.
- **resident** — καρτέλα, υπόλοιπα, (μελλοντικά) συζήτηση.

Πρόσβαση **membership-based**: κάθε χρήστης έχει doc `/users/{email}` με ρόλο και
κτίρια/διαμερίσματα. Bootstrap admin: `michael@crowdpolicy.com`.

## Οντότητες (Firestore)
- `buildings` — κτίριο: κωδικός, διεύθυνση, περιοχή, διαχειριστής, ΙΒΑΝ/ΑΦΜ,
  **`scales`** (πίνακες χιλιοστών, επεξεργάσιμοι), `billingFeePerApartment`
  (έκδοση λογ/σμών), `heatingClosedPercent` (κλειστά %).
- `apartments` — διαμέρισμα: κωδικός, Α/Α, ιδιοκτήτης, ένοικος, `closed`,
  **`millesimes`** (map: scaleKey → χιλιοστά).
- `people` — ιδιοκτήτες/ένοικοι.
- `expenses` — δαπάνη περιόδου: ομάδα (στήλη), κατηγορία, ποσό, μέθοδος
  επιμερισμού (`millesime`/`equal`/`fixedPerApartment`/`heating`), scaleKey.
- `payments` — εισπράξεις ανά διαμέρισμα.
- `statements` — έκδοση κοινοχρήστων (snapshot: rows ανά διαμέρισμα, χιλιοστά,
  ποσά ανά ομάδα, μεταφορά υπολοίπου, σύνολα, expenseLines).
- `fundEntries` — κινήσεις ταμείου/αποθεματικού.
- `users` — πρόσβαση/ρόλοι.
- `auditLogs` — ιστορικό ενεργειών.
- (Φάση 3) `documents`, `announcements`, `topics`, `offers`, `polls`, `votes`, `assemblies`.

## Μηχανή επιμερισμού
Αναπαράγει τη «Συγκεντρωτική Κατάσταση Δαπανών»:
- **Κατά χιλιοστά**: `share = amount × (χιλιοστά[scale] / Σχιλιοστά[scale])`.
- **Ισόποσα**: `amount / N`.
- **Πάγιο ανά διαμέρισμα**: `amount` σε κάθε διαμέρισμα.
- **Θέρμανση**: κατά χιλιοστά θέρμανσης· κλειστά διαμερίσματα πληρώνουν
  `heatingClosedPercent%` του βάρους τους (το υπόλοιπο επιμερίζεται στα ανοιχτά).
- **Έκδοση λογ/σμών**: πάγιο ανά διαμέρισμα (building setting).
- **Σύνολο περιόδου** ανά διαμέρισμα = Σ(ομάδες) + έκδοση.
- **Γενικό σύνολο** ανά διαμέρισμα = προηγ. υπόλοιπο + σύνολο περιόδου.

Επαληθευμένο με πραγματικό έντυπο (Κ. Καραμανλή 17): π.χ. ΠΑΡΙΔΗΣ 86 χιλ. →
κοινόχρηστα 137,47 €, ασανσέρ 32,17 €· γενικό σύνολο 2.259,36 €.

## Seed
Πολυκατοικία-δείγμα «Κ. Καραμανλή 17, Ξυλόκαστρο» με 15 διαμερίσματα, τους δύο
πίνακες χιλιοστών (γενικά/ασανσέρ) και παράδειγμα δαπανών περιόδου.

## Επεκτασιμότητα
Όλα τα έγγραφα φέρουν `buildingId`. Μετάβαση σε πολλαπλές πολυκατοικίες (ή σε
ξεχωριστό GCP project) = migration/config, όχι επανασχεδιασμός.
